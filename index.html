Elevating Drone Safety: Redesigning Altitude Visualization

A UX Case Study by Rishik JHA

Overview

As a UX designer on the Ground Control Station (GCS) team, I led the redesign of our altitude visualization system to prevent drone crashes and enhance operator safety. This project impacted 10,000+ drone operators globally and reduced altitude-related incidents by 85%.

The Challenge

Problem Space

Drone operators were struggling with a critical safety issue: they couldn't easily visualize their drone's height relative to the terrain, leading to dangerous near-miss incidents and reduced operational confidence.

68%

of operators reported difficulty in validating safe flight paths

23%

of flight plans had potential terrain conflicts

15-20 min

User Research

  • 12 contextual inquiries with drone operators
  • 8 stakeholder interviews
  • Analysis of 200+ incident reports
  • 5 competitive analysis sessions

Key insight: Operators needed real-time, visual confirmation of safe altitude margins, not just numerical data.

Design Process

1. Discovery & Definition

I created user journey maps and identified three critical moments:

  • Pre-flight planning
  • Real-time flight monitoring
  • Emergency response scenarios

2. Design Solution

Intuitive Visualization

  • Split-screen interface
  • Color-coded safety zones
  • Interactive elevation profiles

Smart Safety System

  • Predictive terrain warnings
  • One-click altitude adjustment
  • Emergency override protocols

Design Iteration Process

The development of our elevation visualization went through multiple iterations, each responding to specific user needs and feedback. Here's how we evolved the design:

Iteration 1: Basic Concept

User Feedback:

  • "Need clearer distinction between ground and flight path"
  • "Can't easily see height differences"
  • "Missing important measurements"

Iteration 2: Added AGL Visualization

User Feedback:

  • "Better visibility of height differences"
  • "Need more precise measurements"
  • "Dark background would be better for contrast"

Iteration 3: Enhanced Readability

User Feedback:

  • "Much better contrast and readability"
  • "Need hover interactions for precise values"
  • "Could use more detailed grid system"

Final Version: Complete Visualization

Final Features:

  • Interactive tooltips for precise measurements
  • Comprehensive grid system for better reference
  • Clear color coding and legend
  • Optimal contrast for readability

Key Design Decisions

Color Scheme

Dark background with high-contrast elements for better visibility in various lighting conditions.

Interactive Elements

Hover states and tooltips provide precise measurements while maintaining a clean interface.

Visual Hierarchy

Critical information (AGL values) prominently displayed with supporting grid system for context.
